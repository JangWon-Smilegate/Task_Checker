# AI 회의록 자동 생성 시스템 기획서

---

# Part 1. 개요

## 배경

현재 회의가 끝나면 참석자가 직접 회의록을 작성하고 있습니다. 회의 내용을 기억에 의존해 정리하다 보니 30분~1시간이 소요되고, 내용이 누락되거나 왜곡되는 경우가 발생합니다. 작성된 회의록을 관련자에게 공유하는 것도 당일~익일까지 지연되며, 회의에서 나온 할 일(Action Item)이 별도로 추적되지 않아 후속 진행이 빠지는 문제가 있습니다.

## 목적

이 시스템은 **회의 녹음 파일을 업로드하면 AI가 자동으로 회의록을 만들어 주는 것**을 목표로 합니다.

구체적으로, 음성 파일을 넣으면 다음 5가지가 자동으로 처리됩니다:

1. **녹취록 생성** — 음성을 텍스트로 변환하고, 누가 말했는지 화자를 구분합니다.
2. **회의 요약** — 녹취록을 AI가 분석하여 논의 사항, 결정 사항, 미결 사항으로 정리합니다.
3. **To-Do List 생성** — 회의 중 나온 할 일을 자동으로 추출하고, 담당자와 마감일을 매핑합니다.
4. **Task 생성** — 추출된 할 일을 Hansoft 또는 Jira에 Task로 등록합니다. (사용자 확인 후)
5. **공유** — 완성된 회의록을 Confluence에 게시하고, Slack과 이메일로 관련자에게 발송합니다.

## 기대 효과

| 기존 | 개선 후 |
|---|---|
| 회의 후 수동으로 회의록 작성 (30분~1시간) | 음성 업로드 → 자동 생성 (5분 이내) |
| 회의 내용 누락/왜곡 발생 | 전체 발화 기반 정확한 기록 |
| To-Do 추적 누락 | 자동 추출 및 Task 생성으로 추적 보장 |
| 공유 지연 (당일~익일) | 생성 즉시 자동 공유 |

## 동작 방식

사용자가 회의 녹음 파일(mp3, wav, m4a 등)을 업로드하면, 시스템이 아래 순서로 처리합니다:

```
음성 파일 업로드
    ↓
[1] STT 변환 — 음성을 텍스트로 변환하고 화자를 분리합니다
    ↓
[2] AI 분석 — LLM이 녹취록을 읽고 주제별로 논의/결정/미결 사항을 정리합니다
    ↓
[3] To-Do 추출 — "~해야 한다", "~까지 완료" 등의 패턴에서 할 일을 추출합니다
    ↓
[4] 사용자 확인 — 생성된 회의록과 Task 목록을 검토하고 수정/승인합니다
    ↓
[5] 자동 공유 — Confluence 게시, Slack 발송, 이메일 발송, Hansoft/Jira Task 생성
```

## 사용 예시

**Claude Code에서 사용하는 경우:**
```
사용자: "회의 녹음 파일 분석해서 회의록 만들어 줘" (파일 첨부)

Claude:
  1. 녹취록을 생성합니다
  2. 회의 요약을 정리합니다
  3. To-Do 3건을 추출했습니다
  4. "Confluence에 게시하고 Slack으로 공유할까요?"
  5. 사용자 승인 → 자동 게시/공유 완료
```

**웹 인터페이스에서 사용하는 경우:**
```
1. 웹 페이지에서 음성 파일을 드래그 앤 드롭
2. 처리 상태 표시 (업로드 → 변환 중 → 분석 중 → 완료)
3. 생성된 회의록을 미리보고 편집
4. "게시" 버튼 → Confluence + Slack + 이메일 동시 발송
```

# Part 2. 기능 상세

## 1. 녹취록 생성 (Speech-to-Text)

**입력:** 회의 녹음 파일 (mp3, wav, m4a, webm 등)

**처리 과정:**
1. 음성 파일 업로드
2. STT(Speech-to-Text) 엔진으로 음성 → 텍스트 변환
3. 화자 분리 (Speaker Diarization) — 누가 말했는지 구분
4. 타임스탬프 부여

**출력 예시:**
```
[00:00:12] 김상현: 오늘 회의 안건은 원정대 시스템 킥오프입니다.
[00:00:25] 정기영: 네, 개발실에서는 기본 설계안을 준비했습니다.
[00:01:03] 고원준: UI 쪽은 정보창 레이아웃 초안을 공유드릴게요.
...
```

**기술 스택 후보:**

| 엔진 | 특징 | 비고 |
|---|---|---|
| OpenAI Whisper | 다국어 지원, 높은 정확도 | 로컬 실행 가능 |
| Google Speech-to-Text | 실시간 스트리밍 지원, 화자 분리 내장 | 클라우드 API |
| Azure Speech Service | 화자 분리 우수, 기업 환경 적합 | 클라우드 API |
| Clova Speech (네이버) | 한국어 특화, 높은 한국어 정확도 | 클라우드 API |

**고려 사항:**
- 사내망 환경에서의 보안 요건 (외부 API 사용 가능 여부)
- 한국어 인식 정확도 우선
- 화자 분리 정확도 (회의 참여자 수에 따른 성능)

## 2. 회의 요약 (AI Summary)

**입력:** 녹취록 텍스트

**처리 과정:**
1. 녹취록을 LLM(Claude/GPT)에 전달
2. 회의 메타정보 추출 (일시, 장소, 참석자)
3. 주제별 논의 사항 구조화
4. 결정 사항 / 미결 사항 분리

**출력 형식:**
```markdown
# YYYY-MM-DD 회의명

## 회의 정보
| 항목 | 내용 |
|---|---|
| 일시 | 2026년 4월 13일(월) 오전 11시 |
| 장소 | 플레체 |
| 참석자 | 기획실: 김상현, 고원준 / 개발실: 정기영, 조현건 |

## 논의 사항
### 1. [주제 1]
- 논의 내용 요약
- 결정 사항: ...
- 미결 사항: ...

### 2. [주제 2]
- 논의 내용 요약

## 결정 사항
1. ...
2. ...

## 후속 진행 (Action Items)
| 담당 | 내용 | 마감 |
|---|---|---|
| 기획실 | 피드백 반영 기획서 업데이트 | 04-17(금) |
| 개발실 | 프로토타입 구현 | 04-24(금) |
```

---

## 3. To-Do List 생성

**입력:** 회의 요약에서 추출된 Action Items

**처리 과정:**
1. 회의 내용에서 Action Item 자동 추출
   - "~해야 한다", "~진행 예정", "~까지 완료", "~검토 필요" 등 패턴 인식
   - 담당자/부서 매핑
   - 마감일 추출 (언급된 경우)
2. 우선순위 자동 판단 (긴급/높음/보통/낮음)

**출력 형식:**
```markdown
## To-Do List

- [ ] [기획실 김상현] 피드백 반영 기획서 업데이트 (~04-17)
- [ ] [개발실 정기영] 프로토타입 구현 및 내부 테스트 (~04-24)
- [ ] [개발관리실 김유석] QA 테스트 일정 확정 (~04-15)
- [ ] [전체] 차주 월요일 후속 회의 참석
```

## 4. Task 생성 (Hansoft / Jira 연동)

**입력:** To-Do 항목

**처리 과정:**
1. To-Do 항목을 Task 생성 후보 목록으로 표시
2. 사용자 확인/수정 후 생성 승인
3. Hansoft 또는 Jira에 Task 자동 생성

**Hansoft Task 생성:**
```
도구: lam-dashboard API (신규 개발 필요)
필드:
  - 제목: To-Do 내용
  - 담당자: 추출된 담당자
  - 마감일: 추출된 마감일
  - 설명: 회의록 링크 + 관련 논의 내용
  - 우선순위: 자동 판단값
```

**Jira Issue 생성:**
```
도구: mcp__atlassian__jira_create_issue
필드:
  - project: LAM 프로젝트
  - summary: To-Do 내용
  - assignee: 담당자
  - duedate: 마감일
  - description: 회의록 링크 + 관련 논의 내용
  - issuetype: Task
```

**사용자 확인 플로우:**
```
AI: "회의에서 추출된 Task 3건입니다. 생성할까요?"

| # | 담당자 | 내용 | 마감 | 생성 대상 |
|---|---|---|---|---|
| 1 | 김상현 | 피드백 반영 기획서 업데이트 | 04-17 | Hansoft |
| 2 | 정기영 | 프로토타입 구현 | 04-24 | Jira |
| 3 | 김유석 | QA 테스트 일정 확정 | 04-15 | Hansoft |

사용자: "1, 3번만 생성해 줘"
AI: → Hansoft Task 2건 생성 완료
```

## 5. 공유 (메신저 및 이메일)

**공유 채널:**

| 채널 | 방식 | 내용 |
|---|---|---|
| Confluence | 자동 페이지 생성 | 전체 회의록 (녹취록 + 요약 + To-Do) |
| Slack | 메시지 발송 | 회의 요약 + Confluence 링크 |
| 이메일 | 메일 발송 | 회의 요약 + To-Do + Confluence 링크 |

**Confluence 자동 게시:**
```
도구: mcp__atlassian__confluence_create_page
필드:
  - space_key: "LAM"
  - title: "YYYY-MM-DD 회의명"
  - parent_id: 회의록 상위 페이지 ID
  - body: 생성된 회의록 전체 내용 (마크다운 → HTML 변환)
```

**Slack 공유 메시지 형식:**
```
📋 회의록이 생성되었습니다

🗓 2026-04-13 원정대 시스템 킥오프 회의
👥 참석: 김상현, 정기영, 고원준 외 5명

📌 결정 사항:
  1. 원정대 시스템 기본 설계안 확정
  2. 프로토타입 4월 내 구현 목표

📝 To-Do: 3건
🔗 전체 회의록: https://wiki.sgr.com/pages/viewpage.action?pageId=XXXXX
```

**이메일 공유:**
- 수신자: 회의 참석자 전원 (자동) + 추가 수신자 (수동 지정)
- 제목: `[회의록] YYYY-MM-DD 회의명`
- 본문: 회의 요약 + To-Do List + Confluence 링크
